## Execution Guidance

Use this order when continuing refactors:

1. Prefer safety fixes with clear boundaries first: UI process cleanup, lifecycle disposal, and stale API cleanup.
2. For behavior-heavy code (`FormattingService`, RDA parsing, template merging), make one extraction at a time and build after each change.
3. Do not restart completed facade-wrapper cleanup unless a wrapper or dead member reappears; `AssetProcessor.cs` is intentionally just shared state plus `Main` now.
4. `PipelineContext` migration is complete. Do not restart it unless removing the final `Program.Context` bridge.
5. For RDA code, treat archive parsing as high-risk: keep parsing order, validation behavior, and extraction output stable unless adding tests first.
6. For UI localization/logging, preserve existing user-visible strings unless the change explicitly targets localization behavior.
7. `PipelineLogger` rollout is complete across all 9 backend files. All new debug logging should use `context.Log.Debug/Write`.
8. All 39 `AssetProcessorConsole.WriteColoredMessage` calls in the backend are addressed. New console output should use `context.Log.Write(type, msg)` for debug-only or `context.Log.Write(type, msg, always: true)` for always-show messages.
